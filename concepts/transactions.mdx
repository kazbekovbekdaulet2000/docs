Транзакция — это запись о движении средств: оплата, возврат, депозит или ручная корректировка. Такие записи связывают аренду или продажу с фактическими платежами и формируют аудит для биллинга.

## Типы

| Тип | Описание |
|-----|----------|
| `payment` | Входящий платёж по счёту |
| `refund` | Возврат клиенту |
| `deposit` | Залог / предоплата |
| `adjustment` | Ручная корректировка без прямого платежа |

## Поля

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Идентификатор |
| `type` | string | Один из типов выше |
| `amount` | number | Сумма (положительное число) |
| `currency` | string | Код валюты ISO 4217 |
| `status` | string | Например `pending`, `completed`, `failed`, `reversed` |
| `rental_id` | string \| null | Связь с арендой, если есть |
| `created_at` | string | Время создания (ISO 8601) |

## Статусы

| Статус | Описание |
|--------|----------|
| `pending` | Инициировано, нет финального подтверждения |
| `completed` | Успешно проведено |
| `failed` | Не выполнено (ошибка шлюза и т.д.) |
| `reversed` | Отменено после проведения |

<Info>
  Завершённые транзакции не переписывают — для исправлений создают новую запись (возврат, корректировку).
</Info>

## Связь с арендой и биллингом

- Транзакция может ссылаться на `rental_id`.
- Оплаты уменьшают баланс счёта в [биллинге](/concepts/billing).
- Депозиты могут удерживаться или возвращаться при закрытии аренды.
