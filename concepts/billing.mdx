Биллинг — это выставление счетов и учёт оплат по аренде и услугам. В API Yume Cloud счета и карты описаны в разделе **`/v1/billing/...`** (см. OpenAPI).

Связь с деньгами идёт через [транзакции](/concepts/transactions): каждая оплата фиксируется отдельной записью.

## Цикл

- Итоговый счёт может формироваться при завершении аренды.
- Для длительных договоров возможен периодический выставление счетов (по настройкам тенанта).

<Tip>
  Перед закрытием аренды проверьте тариф и реквизиты — итоговый счёт строится на их основе.
</Tip>

## Сущности

### Счёт (invoice)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Идентификатор |
| `rental_id` | string | Связанная аренда |
| `client_id` | string | Плательщик |
| `amount` | number | Сумма к оплате |
| `status` | string | `draft`, `sent`, `paid`, `overdue` и др. |
| `due_date` | string | Срок оплаты |

#### Статусы счёта

| Статус | Описание |
|--------|----------|
| `draft` | Черновик, можно менять |
| `sent` | Отправлен клиенту |
| `paid` | Оплачен полностью |
| `overdue` | Просрочен |

### Оплата

Каждая оплата — это [транзакция](/concepts/transactions) типа `payment`, уменьшающая баланс счёта.

### Баланс

Остаток к оплате = сумма счёта минус сумма завершённых платежей по нему.

## Запросы к API

Используйте пути из спецификации, например:

- списки и детали счетов: `/v1/billing/invoices/`
- расчёты и проверки — см. операции `calculate`, `check`, `result` в OpenAPI

Пример (путь уточняйте по спецификации):

```bash
curl --request GET \
  'https://api.yume.cloud/v1/billing/invoices/?page=1&pageSize=20' \
  --header 'Authorization: Bearer <access>'
```

Параметры фильтрации и тело запросов для ручных операций смотрите в актуальном OpenAPI.
