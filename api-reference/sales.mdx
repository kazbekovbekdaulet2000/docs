---
title: "Sales"
description: "API reference for the Sales resource."
---

The Sales resource manages sales orders — from creation through fulfillment or cancellation. Each sale can include multiple line items, each referencing a product with a quantity and price. All endpoints require a valid Bearer token in the `Authorization` header.

**Base URL:** `https://api.yume.cloud`

---

## List all sales

<ParamField query="client_id" type="string">
  Return sales belonging to a specific client.
</ParamField>

<ParamField query="status" type="string">
  Filter by sale status. Accepted values: `pending`, `completed`, `cancelled`.
</ParamField>

<ParamField query="from_date" type="string">
  Return sales created on or after this date. ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField query="to_date" type="string">
  Return sales created on or before this date. ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for paginated results.
</ParamField>

<ParamField query="per_page" type="number" default="25">
  Number of results per page. Maximum: `100`.
</ParamField>

```bash cURL
curl --request GET \
  --url 'https://api.yume.cloud/sales?status=pending&page=1&per_page=25' \
  --header 'Authorization: Bearer <token>'
```

## Create a sale

<ParamField body="client_id" type="string" required>
  ID of the client making the purchase.
</ParamField>

<ParamField body="items" type="object[]" required>
  Array of line items included in the sale. Each item must include `product_id`, `quantity`, and `price`.

  <Expandable title="item properties">
    <ParamField body="items[].product_id" type="string" required>
      ID of the product being sold.
    </ParamField>
    <ParamField body="items[].quantity" type="number" required>
      Number of units sold. Must be a positive integer.
    </ParamField>
    <ParamField body="items[].price" type="number" required>
      Unit price at the time of sale. Used to calculate the line total.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="notes" type="string">
  Optional free-text notes attached to the sale record.
</ParamField>

```bash cURL
curl --request POST \
  --url https://api.yume.cloud/sales \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "client_id": "cli_93KP0L7D",
    "items": [
      {
        "product_id": "prd_11AA3X2B",
        "quantity": 2,
        "price": 49.99
      },
      {
        "product_id": "prd_22BB4Y3C",
        "quantity": 1,
        "price": 199.00
      }
    ],
    "notes": "Priority order for key account."
  }'
```

## Update sale status

Update the status of an existing sale.

<ParamField path="id" type="string" required>
  The unique identifier of the sale to update.
</ParamField>

<ParamField body="status" type="string" required>
  New status for the sale. Accepted values: `pending`, `completed`, `cancelled`.
</ParamField>

<Tip>
  Transition sales from `pending` to `completed` once payment is confirmed and goods have been dispatched. Use `cancelled` to void a sale without deleting the record.
</Tip>

```bash cURL
curl --request PUT \
  --url https://api.yume.cloud/sales/sal_05OR8U6DQ \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "status": "completed"
  }'
```


## Response fields

<ResponseField name="id" type="string" required>
  Unique identifier for the sale record.
</ResponseField>

<ResponseField name="client_id" type="string" required>
  ID of the client associated with this sale.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current status of the sale: `pending`, `completed`, or `cancelled`.
</ResponseField>

<ResponseField name="items" type="object[]" required>
  Array of line items included in the sale.

  <Expandable title="item properties">
    <ResponseField name="product_id" type="string" required>
      ID of the product sold.
    </ResponseField>
    <ResponseField name="quantity" type="number" required>
      Number of units sold.
    </ResponseField>
    <ResponseField name="price" type="number" required>
      Unit price at the time of sale.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number" required>
  Sum of all line item totals (`quantity × price`) for the sale.
</ResponseField>

<ResponseField name="notes" type="string">
  Optional free-text notes attached to the sale.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the sale was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the most recent update to the sale.
</ResponseField>
