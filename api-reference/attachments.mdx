---
title: "Attachments"
description: "API reference for the Attachments resource."
---

The Attachments resource lets you upload and manage files associated with rentals, sales, and clients — such as photos, signed contracts, or identification documents.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/attachments` | List attachments for a resource |
| `POST` | `/attachments` | Upload an attachment |
| `GET` | `/attachments/{id}` | Get attachment metadata and URL |
| `DELETE` | `/attachments/{id}` | Delete an attachment |

---

## List attachments

`GET /attachments`

Returns all attachments associated with a specific resource.

### Query parameters

<ParamField query="resource_type" type="string" required>
  The type of resource to list attachments for. One of `rental`, `sale`, or `client`.
</ParamField>

<ParamField query="resource_id" type="string" required>
  The ID of the resource to list attachments for.
</ParamField>

### Example request

```bash cURL
curl --request GET \
  --url 'https://api.yume.cloud/attachments?resource_type=rental&resource_id=ren_01HZ8M1A2BCDE3FGHIJ4KL' \
  --header 'Authorization: Bearer <token>'
```

### Example response

```json
{
  "data": [
    {
      "id": "att_01HZA3N4OPQRS5TUVWX6YZ",
      "name": "signed-contract.pdf",
      "resource_type": "rental",
      "resource_id": "ren_01HZ8M1A2BCDE3FGHIJ4KL",
      "url": "https://storage.yume.cloud/attachments/signed-contract.pdf?token=...",
      "size": 204800,
      "created_at": "2024-06-01T11:00:00Z"
    }
  ],
  "page": 1,
  "per_page": 20,
  "total": 1
}
```

## Upload an attachment

`POST /attachments`

Uploads a file and associates it with a resource. This endpoint accepts `multipart/form-data`.

<Note>
  Use `Content-Type: multipart/form-data` for this endpoint. JSON request bodies are not supported.
</Note>

<Warning>
  Only the following file types are supported: PDF, JPG, PNG, WEBP, HEIC, DOCX, XLSX. Uploading unsupported file types will return a `422 Unprocessable Entity` error.
</Warning>

### Form fields

<ParamField body="file" type="binary" required>
  The file to upload. Maximum size is 50 MB.
</ParamField>

<ParamField body="resource_type" type="string" required>
  The type of resource to attach the file to. One of `rental`, `sale`, or `client`.
</ParamField>

<ParamField body="resource_id" type="string" required>
  The ID of the resource to attach the file to.
</ParamField>

<ParamField body="name" type="string">
  A display name for the attachment. If omitted, the original filename is used.
</ParamField>

### Example request

```bash cURL
curl --request POST \
  --url https://api.yume.cloud/attachments \
  --header 'Authorization: Bearer <token>' \
  -F 'file=@/path/to/signed-contract.pdf' \
  -F 'resource_type=rental' \
  -F 'resource_id=ren_01HZ8M1A2BCDE3FGHIJ4KL' \
  -F 'name=Signed contract'
```

### Example response

```json
{
  "id": "att_01HZA3N4OPQRS5TUVWX6YZ",
  "name": "Signed contract",
  "resource_type": "rental",
  "resource_id": "ren_01HZ8M1A2BCDE3FGHIJ4KL",
  "url": "https://storage.yume.cloud/attachments/att_01HZA3N4OPQRS5TUVWX6YZ.pdf?token=...",
  "size": 204800,
  "created_at": "2024-06-01T11:00:00Z"
}
```

## Delete an attachment

`DELETE /attachments/{id}`

Permanently deletes an attachment. This action cannot be undone.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the attachment to delete.
</ParamField>

### Example request

```bash cURL
curl --request DELETE \
  --url https://api.yume.cloud/attachments/att_01HZA3N4OPQRS5TUVWX6YZ \
  --header 'Authorization: Bearer <token>'
```

### Example response

```json
{
  "message": "Attachment deleted successfully."
}
```
