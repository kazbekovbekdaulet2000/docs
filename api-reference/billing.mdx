---
title: "Billing"
description: "API reference for the Billing resource."
---

The Billing resource manages invoices, payments, and outstanding balances. Invoices are generated automatically from rental records or created manually. All endpoints require a valid Bearer token in the `Authorization` header.

**Base URL:** `https://api.yume.cloud`

---

## List invoices

<ParamField query="rental_id" type="string">
  Return invoices linked to a specific rental.
</ParamField>

<ParamField query="client_id" type="string">
  Return invoices belonging to a specific client.
</ParamField>

<ParamField query="status" type="string">
  Filter by invoice status. Accepted values: `draft`, `sent`, `paid`, `overdue`.
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for paginated results.
</ParamField>

<ParamField query="per_page" type="number" default="25">
  Number of results per page. Maximum: `100`.
</ParamField>

```bash cURL
curl --request GET \
  --url 'https://api.yume.cloud/billing?status=overdue&client_id=cli_93KP0L7D' \
  --header 'Authorization: Bearer <token>'
```

```json Response
{
  "data": [
    {
      "id": "inv_14CZ3X0GT",
      "rental_id": "rnt_01HX4K9Z2M",
      "client_id": "cli_93KP0L7D",
      "status": "overdue",
      "amount_due": 500.00,
      "currency": "USD",
      "due_date": "2026-03-31",
      "created_at": "2026-03-01T09:00:00Z",
      "updated_at": "2026-04-01T00:00:00Z"
    }
  ],
  "meta": {
    "page": 1,
    "per_page": 25,
    "total": 3
  }
}
```

## Get an invoice

Retrieve a single invoice by its ID, including all associated line items.

<ParamField path="id" type="string" required>
  The unique identifier of the invoice.
</ParamField>

```bash cURL
curl --request GET \
  --url https://api.yume.cloud/billing/inv_14CZ3X0GT \
  --header 'Authorization: Bearer <token>'
```

```json Response
{
  "id": "inv_14CZ3X0GT",
  "rental_id": "rnt_01HX4K9Z2M",
  "client_id": "cli_93KP0L7D",
  "status": "overdue",
  "amount_due": 500.00,
  "currency": "USD",
  "due_date": "2026-03-31",
  "line_items": [
    {
      "description": "Vehicle rental — April 1–15",
      "quantity": 15,
      "unit": "day",
      "unit_price": 30.00,
      "total": 450.00
    },
    {
      "description": "Late return fee",
      "quantity": 1,
      "unit": "flat",
      "unit_price": 50.00,
      "total": 50.00
    }
  ],
  "created_at": "2026-03-01T09:00:00Z",
  "updated_at": "2026-04-01T00:00:00Z"
}
```

### Invoice response fields

<ResponseField name="id" type="string" required>
  Unique identifier for the invoice.
</ResponseField>

<ResponseField name="rental_id" type="string">
  ID of the linked rental, if applicable.
</ResponseField>

<ResponseField name="client_id" type="string" required>
  ID of the client this invoice is billed to.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current invoice status: `draft`, `sent`, `paid`, or `overdue`.
</ResponseField>

<ResponseField name="amount_due" type="number" required>
  Total outstanding amount on the invoice, in the specified currency.
</ResponseField>

<ResponseField name="currency" type="string" required>
  ISO 4217 currency code for the invoice.
</ResponseField>

<ResponseField name="due_date" type="string" required>
  Payment due date in ISO 8601 format: `YYYY-MM-DD`.
</ResponseField>

<ResponseField name="line_items" type="object[]" required>
  Array of itemized charges included in the invoice.

  <Expandable title="line item properties">
    <ResponseField name="description" type="string" required>
      Human-readable label for this charge.
    </ResponseField>
    <ResponseField name="quantity" type="number" required>
      Number of units charged.
    </ResponseField>
    <ResponseField name="unit" type="string" required>
      Unit of measure for this line item (e.g., `day`, `hour`, `flat`).
    </ResponseField>
    <ResponseField name="unit_price" type="number" required>
      Price per unit.
    </ResponseField>
    <ResponseField name="total" type="number" required>
      Line total calculated as `quantity × unit_price`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the invoice was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the most recent update to the invoice.
</ResponseField>

## Record a payment

Record a payment against an existing invoice. A successful payment reduces the `amount_due` on the invoice and may transition its status to `paid`.

<ParamField body="invoice_id" type="string" required>
  ID of the invoice this payment applies to.
</ParamField>

<ParamField body="amount" type="number" required>
  Amount being paid. Must be a positive decimal number. Partial payments are accepted.
</ParamField>

<ParamField body="currency" type="string" required>
  ISO 4217 currency code for the payment. Must match the invoice currency.
</ParamField>

<ParamField body="method" type="string" required>
  Payment method used. Accepted values: `card`, `cash`, `transfer`.
</ParamField>

<Warning>
  You cannot record a payment against an invoice with status `draft`. Send the invoice first to transition it to `sent`, then record payments against it.
</Warning>

```bash cURL
curl --request POST \
  --url https://api.yume.cloud/billing/payments \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "invoice_id": "inv_14CZ3X0GT",
    "amount": 500.00,
    "currency": "USD",
    "method": "transfer"
  }'
```

```json Response
{
  "id": "pmt_21DW4Y1HU",
  "invoice_id": "inv_14CZ3X0GT",
  "amount": 500.00,
  "currency": "USD",
  "method": "transfer",
  "created_at": "2026-04-09T17:00:00Z"
}
```

### Payment response fields

<ResponseField name="id" type="string" required>
  Unique identifier for the payment record.
</ResponseField>

<ResponseField name="invoice_id" type="string" required>
  ID of the invoice the payment was applied to.
</ResponseField>

<ResponseField name="amount" type="number" required>
  Amount paid.
</ResponseField>

<ResponseField name="currency" type="string" required>
  ISO 4217 currency code for the payment.
</ResponseField>

<ResponseField name="method" type="string" required>
  Payment method used: `card`, `cash`, or `transfer`.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the payment was recorded.
</ResponseField>

## Get outstanding balance

Returns the total outstanding balance for a client across all unpaid and overdue invoices.

<ParamField query="client_id" type="string" required>
  ID of the client whose balance you want to retrieve.
</ParamField>

```bash cURL
curl --request GET \
  --url 'https://api.yume.cloud/billing/balance?client_id=cli_93KP0L7D' \
  --header 'Authorization: Bearer <token>'
```

```json Response
{
  "client_id": "cli_93KP0L7D",
  "outstanding_balance": 750.00,
  "currency": "USD",
  "invoice_count": 3,
  "oldest_due_date": "2026-02-28"
}
```

### Balance response fields

<ResponseField name="client_id" type="string" required>
  ID of the client the balance applies to.
</ResponseField>

<ResponseField name="outstanding_balance" type="number" required>
  Total unpaid amount across all `sent` and `overdue` invoices for this client, in the account's base currency.
</ResponseField>

<ResponseField name="currency" type="string" required>
  ISO 4217 currency code for the outstanding balance.
</ResponseField>

<ResponseField name="invoice_count" type="number" required>
  Number of invoices contributing to the outstanding balance.
</ResponseField>

<ResponseField name="oldest_due_date" type="string" required>
  Due date of the oldest unpaid invoice, in ISO 8601 format. Use this to prioritize collection actions.
</ResponseField>
