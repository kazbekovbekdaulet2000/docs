---
title: "Notifications"
description: "API reference for the Notifications resource."
---

The Notifications resource lets you retrieve and manage in-app notifications generated by system events such as rental updates, payment alerts, and GPS triggers.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/notifications` | List notifications |
| `GET` | `/notifications/{id}` | Get a notification by ID |
| `PUT` | `/notifications/{id}` | Mark a notification as read or unread |
| `POST` | `/notifications/mark-all-read` | Mark all notifications as read |

---

## List notifications

`GET /notifications`

Returns a paginated list of notifications for the authenticated user.

### Query parameters

<ParamField query="read" type="boolean">
  Filter by read status. Pass `true` to return only read notifications, `false` for unread only. Omit to return all.
</ParamField>

<ParamField query="type" type="string">
  Filter notifications by type (e.g. `rental.created`, `payment.received`). See [supported events](/api-reference/integrations#supported-events) for a full list.
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for paginated results.
</ParamField>

<ParamField query="per_page" type="number" default="20">
  Number of results per page.
</ParamField>

### Example request

```bash cURL
curl --request GET \
  --url 'https://api.yume.cloud/notifications?read=false&page=1&per_page=20' \
  --header 'Authorization: Bearer <token>'
```

### Example response

```json
{
  "data": [
    {
      "id": "ntf_01HZD6Q7RSTUV8WXYZ9ABC",
      "type": "payment.received",
      "message": "A payment of €450.00 was received for rental REN-2024-0041.",
      "read": false,
      "created_at": "2024-06-07T08:00:00Z"
    },
    {
      "id": "ntf_01HZD6R8STUVW9XYZA0BCD",
      "type": "invoice.overdue",
      "message": "Invoice INV-2024-0033 is past its due date.",
      "read": false,
      "created_at": "2024-06-06T18:30:00Z"
    }
  ],
  "page": 1,
  "per_page": 20,
  "total": 5
}
```

## Get a notification

`GET /notifications/{id}`

Returns a single notification by ID.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the notification.
</ParamField>

### Example request

```bash cURL
curl --request GET \
  --url https://api.yume.cloud/notifications/ntf_01HZD6Q7RSTUV8WXYZ9ABC \
  --header 'Authorization: Bearer <token>'
```

### Example response

```json
{
  "id": "ntf_01HZD6Q7RSTUV8WXYZ9ABC",
  "type": "payment.received",
  "message": "A payment of €450.00 was received for rental REN-2024-0041.",
  "read": false,
  "created_at": "2024-06-07T08:00:00Z"
}
```

## Mark as read or unread

`PUT /notifications/{id}`

Updates the read status of a single notification.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the notification to update.
</ParamField>

### Request body

<ParamField body="read" type="boolean" required>
  Set to `true` to mark the notification as read, or `false` to mark it as unread.
</ParamField>

### Example request

```bash cURL
curl --request PUT \
  --url https://api.yume.cloud/notifications/ntf_01HZD6Q7RSTUV8WXYZ9ABC \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{ "read": true }'
```

### Example response

```json
{
  "id": "ntf_01HZD6Q7RSTUV8WXYZ9ABC",
  "type": "payment.received",
  "message": "A payment of €450.00 was received for rental REN-2024-0041.",
  "read": true,
  "created_at": "2024-06-07T08:00:00Z"
}
```

## Mark all as read

`POST /notifications/mark-all-read`

Marks every unread notification for the authenticated user as read in a single request.

### Example request

```bash cURL
curl --request POST \
  --url https://api.yume.cloud/notifications/mark-all-read \
  --header 'Authorization: Bearer <token>'
```

### Example response

```json
{
  "message": "All notifications marked as read.",
  "updated": 5
}
```
