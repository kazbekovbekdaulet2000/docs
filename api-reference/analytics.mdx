---
title: "Analytics"
description: "API reference for the Analytics resource."
---

The Analytics resource provides aggregated reporting on rental activity, revenue, and asset utilization. Use these endpoints to build dashboards, generate reports, or monitor operational health. All endpoints require a valid Bearer token in the `Authorization` header.

**Base URL:** `https://api.yume.cloud`

---

## Get aggregated analytics

Returns time-series analytics data grouped by the specified period and metric type.

<ParamField query="from_date" type="string" required>
  Start of the reporting window. ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField query="to_date" type="string" required>
  End of the reporting window. ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField query="group_by" type="string" default="day">
  Time granularity for grouping results. Accepted values: `day`, `week`, `month`.
</ParamField>

<ParamField query="metric" type="string" default="rentals">
  The metric to aggregate. Accepted values: `rentals`, `revenue`, `utilization`.
</ParamField>

<Tip>
  Use `group_by=month` for high-level trend reporting and `group_by=day` when diagnosing short-term performance changes.
</Tip>

```bash cURL
curl --request GET \
  --url 'https://api.yume.cloud/analytics?from_date=2026-01-01&to_date=2026-03-31&group_by=month&metric=revenue' \
  --header 'Authorization: Bearer <token>'
```

```json Response
{
  "data": [
    {
      "period": "2026-01",
      "total_rentals": 84,
      "total_revenue": 21400.00,
      "utilization_rate": 0.76
    },
    {
      "period": "2026-02",
      "total_rentals": 91,
      "total_revenue": 23150.50,
      "utilization_rate": 0.82
    },
    {
      "period": "2026-03",
      "total_rentals": 78,
      "total_revenue": 19800.00,
      "utilization_rate": 0.70
    }
  ],
  "meta": {
    "from_date": "2026-01-01",
    "to_date": "2026-03-31",
    "group_by": "month",
    "metric": "revenue"
  }
}
```

### Analytics data fields

<ResponseField name="period" type="string" required>
  The time period label for this data point. Format depends on `group_by`: `YYYY-MM-DD` for `day`, `YYYY-[W]WW` for `week`, `YYYY-MM` for `month`.
</ResponseField>

<ResponseField name="total_rentals" type="number" required>
  Total number of rentals that were active or created during this period.
</ResponseField>

<ResponseField name="total_revenue" type="number" required>
  Sum of all completed payment transactions recorded during this period, in your account's base currency.
</ResponseField>

<ResponseField name="utilization_rate" type="number" required>
  Ratio of asset-days rented to total available asset-days during this period. A value of `1.0` means all assets were rented every day. A value of `0.0` means no assets were rented.
</ResponseField>

## Get analytics summary

Returns key metrics for the current billing or calendar period in a single snapshot. Useful for dashboard header cards and health checks.

```bash cURL
curl --request GET \
  --url https://api.yume.cloud/analytics/summary \
  --header 'Authorization: Bearer <token>'
```

```json Response
{
  "active_rentals": 37,
  "monthly_revenue": 9450.00,
  "total_clients": 214,
  "overdue_rentals": 4
}
```

### Summary fields

<ResponseField name="active_rentals" type="number" required>
  Number of rentals currently in `active` status at the time of the request.
</ResponseField>

<ResponseField name="monthly_revenue" type="number" required>
  Total revenue from completed payment transactions in the current calendar month, in your account's base currency.
</ResponseField>

<ResponseField name="total_clients" type="number" required>
  Total number of client records in the system, regardless of rental activity.
</ResponseField>

<ResponseField name="overdue_rentals" type="number" required>
  Number of rentals whose `end_date` has passed but whose status has not been updated to `completed` or `cancelled`. A non-zero value here typically requires operational follow-up.
</ResponseField>
