---
title: "Transactions"
description: "API reference for the Transactions resource."
---

The Transactions resource records all financial movements — payments, refunds, deposits, and adjustments — linked to rentals or other records. All endpoints require a valid Bearer token in the `Authorization` header.

**Base URL:** `https://api.yume.cloud`

<Note>
  Transactions are immutable once their status reaches `completed`. You cannot edit or delete a completed transaction. To correct an error, create a new transaction of type `adjustment` or `refund`.
</Note>

---

## List transactions

<ParamField query="type" type="string">
  Filter by transaction type. Accepted values: `payment`, `refund`, `deposit`, `adjustment`.
</ParamField>

<ParamField query="status" type="string">
  Filter by transaction status. Accepted values: `pending`, `completed`, `failed`.
</ParamField>

<ParamField query="rental_id" type="string">
  Return transactions linked to a specific rental.
</ParamField>

<ParamField query="from_date" type="string">
  Return transactions created on or after this date. ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField query="to_date" type="string">
  Return transactions created on or before this date. ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for paginated results.
</ParamField>

<ParamField query="per_page" type="number" default="25">
  Number of results per page. Maximum: `100`.
</ParamField>

```bash cURL
curl --request GET \
  --url 'https://api.yume.cloud/transactions?type=payment&status=completed&page=1' \
  --header 'Authorization: Bearer <token>'
```

```json Response
{
  "data": [
    {
      "id": "txn_09ZW1V8ER",
      "type": "payment",
      "status": "completed",
      "amount": 250.00,
      "currency": "USD",
      "rental_id": "rnt_01HX4K9Z2M",
      "notes": "Monthly rental payment.",
      "created_at": "2026-04-01T12:00:00Z"
    }
  ],
  "meta": {
    "page": 1,
    "per_page": 25,
    "total": 310
  }
}
```

## Create a transaction

<ParamField body="type" type="string" required>
  Type of transaction. Accepted values: `payment`, `refund`, `deposit`, `adjustment`.
</ParamField>

<ParamField body="amount" type="number" required>
  Transaction amount as a positive decimal number. For refunds, this is the amount returned to the client.
</ParamField>

<ParamField body="currency" type="string" required>
  ISO 4217 currency code (e.g., `USD`, `EUR`, `GBP`).
</ParamField>

<ParamField body="rental_id" type="string">
  ID of the rental this transaction is associated with. Optional — omit for standalone transactions.
</ParamField>

<ParamField body="notes" type="string">
  Optional free-text notes describing the transaction.
</ParamField>

<Tip>
  Always link a transaction to a `rental_id` when applicable. This enables accurate per-rental financial reporting in the Analytics resource.
</Tip>

```bash cURL
curl --request POST \
  --url https://api.yume.cloud/transactions \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "type": "payment",
    "amount": 250.00,
    "currency": "USD",
    "rental_id": "rnt_01HX4K9Z2M",
    "notes": "Monthly rental payment."
  }'
```

```json Response
{
  "id": "txn_10AX2W9FS",
  "type": "payment",
  "status": "completed",
  "amount": 250.00,
  "currency": "USD",
  "rental_id": "rnt_01HX4K9Z2M",
  "notes": "Monthly rental payment.",
  "created_at": "2026-04-09T16:00:00Z"
}
```

## Get a transaction

Retrieve a single transaction by its ID.

<ParamField path="id" type="string" required>
  The unique identifier of the transaction.
</ParamField>

```bash cURL
curl --request GET \
  --url https://api.yume.cloud/transactions/txn_10AX2W9FS \
  --header 'Authorization: Bearer <token>'
```

```json Response
{
  "id": "txn_10AX2W9FS",
  "type": "payment",
  "status": "completed",
  "amount": 250.00,
  "currency": "USD",
  "rental_id": "rnt_01HX4K9Z2M",
  "notes": "Monthly rental payment.",
  "created_at": "2026-04-09T16:00:00Z"
}
```

## Response fields

<ResponseField name="id" type="string" required>
  Unique identifier for the transaction.
</ResponseField>

<ResponseField name="type" type="string" required>
  Transaction type: `payment`, `refund`, `deposit`, or `adjustment`.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current status of the transaction: `pending`, `completed`, or `failed`.
</ResponseField>

<ResponseField name="amount" type="number" required>
  Transaction amount as a positive decimal number.
</ResponseField>

<ResponseField name="currency" type="string" required>
  ISO 4217 currency code for the transaction.
</ResponseField>

<ResponseField name="rental_id" type="string">
  ID of the linked rental, if applicable.
</ResponseField>

<ResponseField name="notes" type="string">
  Optional notes attached to the transaction.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the transaction was created. This value does not change after creation.
</ResponseField>
