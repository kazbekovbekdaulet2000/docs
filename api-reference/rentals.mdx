---
title: "Rentals"
description: "API reference for the Rentals resource."
---

The Rentals resource manages the full lifecycle of a rental — from creation through return or cancellation. All endpoints require a valid Bearer token in the `Authorization` header.

**Base URL:** `https://api.yume.cloud`

---

## List all rentals

<ParamField query="status" type="string">
  Filter by rental status. Accepted values: `active`, `completed`, `cancelled`, `overdue`.
</ParamField>

<ParamField query="client_id" type="string">
  Return rentals belonging to a specific client.
</ParamField>

<ParamField query="asset_id" type="string">
  Return rentals associated with a specific asset.
</ParamField>

<ParamField query="start_date" type="string">
  Filter rentals that start on or after this date. ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField query="end_date" type="string">
  Filter rentals that end on or before this date. ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for paginated results.
</ParamField>

<ParamField query="per_page" type="number" default="25">
  Number of results per page. Maximum: `100`.
</ParamField>

```bash cURL
curl --request GET \
  --url 'https://api.yume.cloud/rentals?status=active&page=1&per_page=25' \
  --header 'Authorization: Bearer <token>'
```

## Create a rental

<ParamField body="asset_id" type="string" required>
  ID of the asset to be rented.
</ParamField>

<ParamField body="client_id" type="string" required>
  ID of the client renting the asset.
</ParamField>

<ParamField body="start_date" type="string" required>
  Rental start date in ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField body="end_date" type="string" required>
  Rental end date in ISO 8601 format: `YYYY-MM-DD`. Must be after `start_date`.
</ParamField>

<ParamField body="notes" type="string">
  Optional free-text notes attached to the rental record.
</ParamField>

```bash cURL
curl --request POST \
  --url https://api.yume.cloud/rentals \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "asset_id": "ast_07BV2R1A",
    "client_id": "cli_93KP0L7D",
    "start_date": "2026-05-01",
    "end_date": "2026-05-10",
    "notes": "Customer requested early return option."
  }'
```

```json Response
{
  "id": "rnt_02LY5M3BN",
  "asset_id": "ast_07BV2R1A",
  "client_id": "cli_93KP0L7D",
  "status": "active",

  "notes": "Customer requested early return option.",
```

## Update a rental

Update the status, end date, or notes of an existing rental.

<ParamField path="id" type="string" required>
  The unique identifier of the rental to update.
</ParamField>

<ParamField body="status" type="string">
  New status for the rental. Accepted values: `active`, `completed`, `cancelled`, `overdue`.
</ParamField>

<ParamField body="end_date" type="string">
  Revised end date in ISO 8601 format: `YYYY-MM-DD`.
</ParamField>

<ParamField body="notes" type="string">
  Updated notes for the rental record.
</ParamField>

```bash cURL
curl --request PUT \
  --url https://api.yume.cloud/rentals/rnt_02LY5M3BN \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "status": "completed",
    "end_date": "2026-05-08",
    "notes": "Returned two days early. No damage."
  }'
```

## Delete a rental

Delete or cancel a rental by its ID.

<ParamField path="id" type="string" required>
  The unique identifier of the rental to delete.
</ParamField>

<Warning>
  Deleting a rental is irreversible. If you want to preserve the record for auditing, set `status` to `cancelled` using the update endpoint instead.
</Warning>

```bash cURL
curl --request DELETE \
  --url https://api.yume.cloud/rentals/rnt_02LY5M3BN \
  --header 'Authorization: Bearer <token>'
```

```json Response
{
  "id": "rnt_02LY5M3BN",
  "deleted": true
}
```

## Response fields

<ResponseField name="id" type="string" required>
  Unique identifier for the rental record.
</ResponseField>

<ResponseField name="asset_id" type="string" required>
  ID of the asset associated with this rental.
</ResponseField>

<ResponseField name="client_id" type="string" required>
  ID of the client who holds this rental.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current status of the rental: `active`, `completed`, `cancelled`, or `overdue`.
</ResponseField>

<ResponseField name="start_date" type="string" required>
  The date the rental began, in ISO 8601 format.
</ResponseField>

<ResponseField name="end_date" type="string" required>
  The date the rental is scheduled to end, in ISO 8601 format.
</ResponseField>

<ResponseField name="notes" type="string">
  Optional free-text notes attached to the rental.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the rental was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the most recent update to the rental.
</ResponseField>
