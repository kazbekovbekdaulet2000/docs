---
title: "Access Rights"
description: "API reference for the Access Rights and user management resource."
---

The Access Rights resource lets you manage which users can access your Yume Cloud account and what they can do. Assign roles to control permissions across the platform.

<Warning>
  Only users with the **Admin** role can manage access rights. Requests from users with any other role will return a `403 Forbidden` error.
</Warning>

## Roles

| Role | Description |
|------|-------------|
| `admin` | Full access to all resources, settings, and user management |
| `manager` | Can manage rentals, sales, and clients, but cannot manage users or billing |
| `operator` | Can view and update assigned rentals and assets |
| `viewer` | Read-only access to all resources |

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/access-rights` | List all users and their roles |
| `POST` | `/access-rights` | Grant access to a user |
| `GET` | `/access-rights/{id}` | Get an access rights entry by ID |
| `PUT` | `/access-rights/{id}` | Update a user's role |
| `DELETE` | `/access-rights/{id}` | Revoke a user's access |

## List access rights

`GET /access-rights`

Returns all users and their assigned roles for the authenticated account.

### Example request

```bash cURL
curl --request GET \
  --url https://api.yume.cloud/access-rights \
  --header 'Authorization: Bearer <token>'
```

### Example response

```json
{
  "data": [
    {
      "id": "acr_01HZH0U1VWXYZ2ABCD4EFG",
      "user_id": "usr_01HZ7L0Z9XYAB1CDEFG2HI",
      "email": "alice@example.com",
      "role": "admin",
      "created_at": "2024-01-15T09:00:00Z"
    },
    {
      "id": "acr_01HZH0V2WXYZ3BCDE5FGH",
      "user_id": "usr_01HZ6K9Y8XWAB0BCDEF1GH",
      "email": "bob@example.com",
      "role": "manager",
      "created_at": "2024-03-01T10:30:00Z"
    }
  ],
  "total": 2
}
```

## Grant access to a user

`POST /access-rights`

Grants a user access to the account with a specified role.

### Request body

<ParamField body="user_id" type="string" required>
  The ID of the user to grant access to.
</ParamField>

<ParamField body="role" type="string" required>
  The role to assign to the user. One of `admin`, `manager`, `operator`, or `viewer`.
</ParamField>

### Example request

```bash cURL
curl --request POST \
  --url https://api.yume.cloud/access-rights \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "user_id": "usr_01HZ5J8X7WVAB9ABCDE0FG",
    "role": "operator"
  }'
```

### Example response

```json
{
  "id": "acr_01HZH0W3XYZA4CDEF6GHI",
  "user_id": "usr_01HZ5J8X7WVAB9ABCDE0FG",
  "email": "charlie@example.com",
  "role": "operator",
  "created_at": "2024-06-07T14:00:00Z"
}
```

## Get an access rights entry

`GET /access-rights/{id}`

Returns a single access rights entry by ID.

### Path parameters

### Example response

```json
  "email": "charlie@example.com",
  "role": "operator",
  "created_at": "2024-06-07T14:00:00Z"

```

## Update a user's role

`PUT /access-rights/{id}`

Updates the role assigned to a user.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the access rights entry to update.
</ParamField>

### Request body

<ParamField body="role" type="string" required>
  The new role to assign. One of `admin`, `manager`, `operator`, or `viewer`.
</ParamField>

### Example request

```bash cURL
curl --request PUT \
  --url https://api.yume.cloud/access-rights/acr_01HZH0W3XYZA4CDEF6GHI \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{ "role": "manager" }'
```

### Example response

```json
{
  "id": "acr_01HZH0W3XYZA4CDEF6GHI",
  "user_id": "usr_01HZ5J8X7WVAB9ABCDE0FG",
  "email": "charlie@example.com",
  "role": "manager",
  "created_at": "2024-06-07T14:00:00Z"
}
```

## Revoke a user's access

`DELETE /access-rights/{id}`

Revokes access for a user by removing their access rights entry. The user will no longer be able to access the account.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the access rights entry to revoke.
</ParamField>

### Example request

```bash cURL
curl --request DELETE \
  --url https://api.yume.cloud/access-rights/acr_01HZH0W3XYZA4CDEF6GHI \
  --header 'Authorization: Bearer <token>'
```

### Example response

```json
{
  "message": "Access revoked successfully."
}
```
