title: "Управление арендой"
description: "Создание, обновление и закрытие аренды через API."
---

Потоки проката в Yume Cloud реализованы через ресурсы CRM и инвентаря (`/v1/crm/...`). Точные пути и поля смотрите в OpenAPI — ниже приведена **концептуальная** последовательность.

<Steps>
  <Step title="Создайте сущности клиента и инвентаря">
    Убедитесь, что в системе есть клиент (`/v1/crm/clients/`) и единица инвентаря/группа (`/v1/crm/inventories/`, `/v1/crm/groups/` — по вашей модели).
  </Step>

  <Step title="Оформите выдачу (сделку)">
    Создайте запись сделки/выдачи согласно вашему процессу (см. соответствующие `POST` в спецификации). Пример заголовков:

    ```bash
    curl --request POST \
      --url 'https://api.yume.cloud/v1/crm/...' \
      --header 'Authorization: Bearer <access>' \
      --header 'Content-Type: application/json' \
      --data '{ ... }'
    ```

    Сохраните `id` из ответа для следующих шагов.
  </Step>

  <Step title="Переведите статус в «активно»">
    Когда объект передан клиенту, обновите статус сущности через `PATCH`/`PUT` на соответствующий ресурс (поля статуса — в схеме OpenAPI).
  </Step>

  <Step title="Получите список активных сделок">
    Используйте `GET` с фильтрами `page`, `pageSize` и параметрами фильтрации из спецификации для нужного списка (клиенты, инвентарь, заказы).

    <Tip>
      Комбинируйте query-параметры, чтобы сузить выборку (например, по клиенту или статусу).
    </Tip>
  </Step>

  <Step title="Завершите аренду">
    Переведите запись в конечный статус (завершено). Обычно это инициирует биллинг/документы — проверьте счета в `/v1/billing/invoices/`.
  </Step>

  <Step title="Отмена">
    При отмене укажите причину в теле запроса, если схема это допускает.

    <Warning>
      При отмене могут применяться штрафы по правилам тенанта — уточните политику у администратора.
    </Warning>
  </Step>
</Steps>

## Где смотреть детали

Полный перечень эндпоинтов — в **OpenAPI** (после `bash scripts/sync-openapi.sh` — полный файл спецификации).
