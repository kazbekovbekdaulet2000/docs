---
title: "GPS tracking"
description: "How to retrieve real-time and historical location data for your assets."
---

Yume Cloud's GPS API lets you query the current and historical location of tracked assets. Use it to monitor where rented equipment is in the field, verify returns, and build location-aware workflows.

<Info>
  GPS data is only available for assets with an active tracker device attached. If an asset has no tracker, location endpoints return a `404` for that asset.
</Info>

<Steps>
  <Step title="Get the current location">
    Retrieve the latest known coordinates for an asset using `GET /gps/locations` with the `asset_id` query parameter.

    <CodeGroup>

    ```bash cURL
    curl --request GET \
      --url 'https://api.yume.cloud/gps/locations?asset_id=asset_xyz789' \
      --header 'Authorization: Bearer <your-access-token>'
    ```

    ```json Response
    {
      "asset_id": "asset_xyz789",
      "lat": 51.5074,
      "lon": -0.1278,
      "timestamp": "2026-04-09T08:45:00Z"
    }
    ```

    </CodeGroup>

    The `timestamp` field reflects when the tracker last reported its position.
  </Step>

  <Step title="Get location history">
    Retrieve a time-bounded sequence of location records by adding `from` and `to` query parameters.

    <CodeGroup>

    ```bash cURL
    curl --request GET \
      --url 'https://api.yume.cloud/gps/locations?asset_id=asset_xyz789&from=2026-04-01&to=2026-04-09' \
      --header 'Authorization: Bearer <your-access-token>'
    ```

    ```json Response
    [
      {
        "asset_id": "asset_xyz789",
        "lat": 51.5080,
        "lon": -0.1290,
        "timestamp": "2026-04-01T07:00:00Z"
      },
      {
        "asset_id": "asset_xyz789",
        "lat": 51.5074,
        "lon": -0.1278,
        "timestamp": "2026-04-05T13:30:00Z"
      },
      {
        "asset_id": "asset_xyz789",
        "lat": 51.5060,
        "lon": -0.1265,
        "timestamp": "2026-04-09T08:45:00Z"
      }
    ]
    ```

    </CodeGroup>

    Records are returned in chronological order. Use the `from` and `to` parameters with ISO 8601 date strings (`YYYY-MM-DD`) to control the time window.
  </Step>

  <Step title="Use location data in rentals">
    GPS location data integrates naturally with the rental lifecycle:

    - **Dispatch verification** — confirm an asset has left your depot before activating a rental.
    - **In-field monitoring** — check that rented assets remain within an expected service area during the rental period.
    - **Return confirmation** — verify an asset's last recorded position matches your return location before marking a rental as `completed`.
    - **Dispute resolution** — use historical location records as an audit trail if a client disputes usage or damage.

    Query `GET /gps/locations?asset_id={id}` at any point during the rental using the asset ID returned in the rental record.
  </Step>
</Steps>
