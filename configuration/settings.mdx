Поведение тенанта и пользовательские предпочтения могут настраиваться через API и интерфейс продукта. Конкретные пути и поля зависят от версии бэкенда — ориентируйтесь на **OpenAPI** (поиск по `settings`, `tenant`, `profile`).

## Типичные группы

- **Глобальная конфигурация** — валюта, часовой пояс, политики аренды (если вынесены в API).
- **Профиль пользователя** — `GET`/`PATCH` **`/v1/auth/profile/`** (см. спецификацию).

## Пример: профиль

```bash
curl --request GET \
  --url https://api.yume.cloud/v1/auth/profile/ \
  --header 'Authorization: Bearer <access>'
```

Обновление — `PUT` или `PATCH` на тот же путь с телом из схемы `ProfileLimited` / `PatchedProfileLimited`.

<Tip>
  Не храните пароли в логах и примерах запросов.
</Tip>
